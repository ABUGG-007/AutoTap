# 自动化点击工具 (AutoClicker) 规格说明书

## 1. 背景与目标

### 1.1 解决的问题
手动重复执行鼠标点击和键盘输入操作非常耗时，特别是对于需要反复执行相同任务的场景（如批量数据录入、自动化测试、表格填写等）。

### 1.2 核心价值
通过记录用户的操作序列，并以可配置的速度自动回放，实现一键自动化执行，大幅节省时间和精力。

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 操作录制
- **开始录制按钮**：点击后进入录制模式，开始监听并记录所有用户输入
- **结束录制按钮**：点击后停止录制，保存完整的操作序列
- **录制状态指示**：实时显示当前是否处于录制状态

#### 2.1.2 支持的操作类型
| 操作类型 | 说明 | 录制内容 |
|---------|------|---------|
| 鼠标左键点击 | 单次左键按下/释放 | 坐标位置(x, y) |
| 鼠标右键点击 | 单次右键按下/释放 | 坐标位置(x, y) |
| 鼠标移动 | 光标位置变化 | 目标坐标(x, y) |
| 键盘按键 | 任意键盘输入 | 按键名称、是否按住Shift/Ctrl/Alt |
| 文本输入 | 连续字符输入 | 完整字符串 |
| 复制操作 | Ctrl+C | 模拟快捷键 |
| 粘贴操作 | Ctrl+V | 模拟快捷键 |
| 上移键 | 滚动或选择上移 | 模拟按键 |
| 下移键 | 滚动或选择下移 | 模拟按键 |

#### 2.1.3 操作回放
- **开始回放按钮**：从当前操作序列的第一条开始执行
- **停止回放按钮**：立即中断回放过程
- **回放进度显示**：显示当前执行到第几个操作

#### 2.1.4 时间控制
- **默认速度模式**：使用录制时的实际时间间隔
- **自定义间隔设置**：用户可设置固定时间间隔（毫秒）
- **速度调节**：提供滑块或倍速选项（0.5x, 1x, 2x, 4x）

#### 2.1.5 操作管理
- **操作序列显示**：在界面上展示已录制的所有操作
- **操作预览**：显示每个操作的详细信息（类型、坐标/内容、时间戳）
- **删除操作**：可删除单个或全部操作
- **导入/导出**：支持保存和加载操作序列文件

### 2.2 辅助功能

#### 2.2.1 系统托盘
- 最小化到系统托盘，不占用任务栏
- 托盘图标显示录制/回放状态
- 托盘菜单快捷操作

#### 2.2.2 全局快捷键
| 功能 | 默认快捷键 |
|-----|----------|
| 开始/停止录制 | F9 |
| 开始/停止回放 | F10 |
| 紧急停止 | Esc |

#### 2.2.3 设置功能
- 开机自启动选项
- 录制/回放时显示通知
- 错误处理策略（点击失败时重试次数）

## 3. 技术选型

### 3.1 推荐技术栈
| 组件 | 技术选择 | 理由 |
|-----|---------|------|
| 编程语言 | Python 3.10+ | 丰富的自动化库、跨平台支持 |
| GUI框架 | PyQt6 | 功能完整、性能良好、外观现代 |
| 输入检测 | pynput | 支持键盘鼠标监听、跨平台 |
| 自动化执行 | pyautogui + pydirectinput | 成熟的鼠标键盘模拟库 |
| 数据存储 | JSON | 轻量、易读、跨语言兼容 |
| 打包分发 | PyInstaller | 一键打包为可执行文件 |

### 3.2 替代方案对比

| 方案 | 优点 | 缺点 |
|-----|------|------|
| PyQt6 + pynput + pyautogui | 功能完整、生态成熟 | 需要安装依赖 |
| Electron + Native Messaging | 界面美观、可网页端控制 | 实现复杂、重量级 |
| AutoHotkey | 轻量、脚本语言简单 | 仅Windows、界面开发弱 |

**最终选择**：Python + PyQt6 + pynput + pyautogui（平衡功能与复杂度）

## 6. 需求规格

### 6.1 录制功能
- **REQ-001**: 系统应提供开始录制按钮，点击后立即开始监听用户输入
- **REQ-002**: 系统应能识别并记录鼠标左键点击的坐标位置
- **REQ-003**: 系统应能识别并记录鼠标右键点击的坐标位置
- **REQ-004**: 系统应能识别并记录键盘按键，包括普通按键和组合键
- **REQ-005**: 系统应能识别复制(Ctrl+C)和粘贴(Ctrl+V)快捷键
- **REQ-006**: 系统应能识别上移键和下移键的按下事件
- **REQ-007**: 每个操作应记录相对于前一个操作的间隔时间（毫秒）
- **REQ-008**: 录制过程中应实时显示操作计数和最新操作类型

### 6.2 回放功能
- **REQ-009**: 系统应提供开始回放按钮，按顺序执行所有录制的操作
- **REQ-010**: 默认回放速度应与录制时的实际间隔一致
- **REQ-011**: 系统应支持用户自定义固定回放间隔
- **REQ-012**: 系统应支持速度倍速调节（0.5x - 4x）
- **REQ-013**: 回放过程中应显示当前执行进度
- **REQ-014**: 系统应提供停止回放按钮，立即中断执行

### 6.3 数据管理
- **REQ-015**: 系统应支持导出操作序列为JSON文件
- **REQ-016**: 系统应支持导入JSON文件恢复操作序列
- **REQ-017**: 系统应支持清空当前操作列表
- **REQ-018**: 系统应支持循环执行，可设置次数或无限循环

### 6.4 系统集成
- **REQ-019**: 系统应提供全局快捷键，在其他应用前台也能响应
- **REQ-020**: 系统应支持最小化到系统托盘
- **REQ-021**: 托盘图标应能反映当前录制/回放状态

### 6.5 配置功能
- **REQ-022**: 系统应支持开机自启动选项
- **REQ-023**: 录制和回放开始时应显示系统通知

## 7. 错误处理

### 7.1 异常场景
| 场景 | 处理策略 |
|-----|---------|
| 点击目标窗口不可用 | 重试3次，间隔500ms，仍失败则提示用户 |
| 录制过程中焦点丢失 | 记录错误，继续录制后续操作 |
| 回放过程中窗口未就绪 | 等待并重试，配置最大等待时间 |
| 快捷键冲突 | 提示用户更改快捷键设置 |

## 8. 性能要求
- 录制响应延迟 < 10ms
- 操作记录内存占用 < 10MB（10000个操作）
- 回放执行精度 ±50ms
- 应用启动时间 < 3秒

## 9. 兼容性
- Windows 10/11（主要目标）
- Python 3.10+
- 分辨率自适应（支持不同DPI设置）

## 10. 发布形式
- Windows 可执行文件 (.exe)
- 无需管理员权限运行（使用用户级API）
